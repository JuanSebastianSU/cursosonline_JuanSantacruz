Documentación del sitio

Estructura semántica

  • Layout base: <header>, <main id="pagina"> (parciales), <footer>.
  • Vistas: secciones con <section>/<article> y jerarquía de h1–h3.
  • Navegación: hash-router simple; menús generados desde API o fallback.

Decisiones de diseño

  • Escala tipográfica del sistema, contraste en modo oscuro, radios y sombras suaves.
  • Grid responsivo (4→3→2→1) y Flex para acciones/controles.
  • Tokens CSS: colores, radios, sombras; clases reutilizables (.card, .badge, etc.).

Técnicas aplicadas

  • DOM: selección por id/class/tag, innerText/innerHTML, estilos dinámicos, creación/eliminación de nodos.
  • Eventos: click, mouseover, keydown, input/submit en formularios.
  • Accesibilidad: aria-label, aria-live, summary/details, sr-only.
  • Rendimiento UX: cache de parciales + AbortController.
